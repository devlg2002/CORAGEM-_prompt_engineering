## O que é o Método Coragem™

A engenharia de prompt convencional é majoritariamente operacional: trata o modelo de linguagem como um sistema de execução instrucional e foca em clareza de comandos. O **Método Coragem™** parte de uma premissa diferente.

Um LLM não executa comandos — ele colapsa distribuições de probabilidade em resposta ao **vetor semântico total do contexto fornecido**. O prompt não é uma ordem: é um campo de condicionamento. Compreender isso muda tudo.

O Método Coragem™ formaliza **9 camadas arquiteturais** que atuam sobre esse campo com precisão filosófica, inferencial e meta-cognitiva — transformando o design de prompts em uma disciplina com teoria própria.

## As 9 Camadas

O Método Coragem™ estrutura qualquer prompt em camadas com funções epistêmicas distintas. As primeiras 8 são universais; a 9ª é ativada quando a complexidade exige múltiplos agentes.

| # | Camada | Natureza | Função |
|---|--------|----------|--------|
| 1 | **Manifesto** | Ontológica | Define o campo gravitacional semântico — o horizonte filosófico antes de qualquer instrução |
| 2 | **Goal** | Teleológica | Vetor de convergência em três dimensões: operacional, epistêmico e filosófico |
| 3 | **Warnings** | Deontológica | Exclusão semântica estruturada — o que o modelo deve ativamente evitar |
| 4 | **Constraints** | Normativa | Fronteiras positivas da inferência — a geometria do espaço de soluções válidas |
| 5 | **Constructs** | Semântica | Vocabulário inferencial e frames de referência que o modelo deve usar como unidades de raciocínio |
| 6 | **Contexto Intrínseco** | Hermenêutica | Horizonte situacional — âncora do raciocínio na realidade específica da tarefa |
| 7 | **Output Esperado** | Morfológica | Forma-alvo da resposta: locução, ilocução e perlocução |
| 8 | **Paramprompting** | Meta-Cognitiva | Modulação dos parâmetros implícitos do processo inferencial — o *como pensar*, não o *o que produzir* |
| 9 | **Config Agêntica** | Heterárquica | Quando aplicável: rede de agentes especializados com retroalimentação entre as três verdades |

### SuperLib™ de Paramprompting (`superlib/`)

Biblioteca de 66 instruções de modulação cognitiva organizadas em 11 modos. Cada instrução especifica *como o modelo deve raciocinar* — não o que deve produzir. É a implementação prática da Camada 8.

**Os 11 modos:**

| Modo | Nome | Objetivo Epistêmico |
|------|------|---------------------|
| I | Exploratório | Expansão do espaço de busca — alta entropia, colapso diferido |
| II | Analítico-Estrutural | Rigor lógico-formal — cadeia causal, falsificação ativa |
| III | Sistêmico | Pensamento interconectado — feedbacks, emergência, segunda ordem |
| IV | Estratégico | Aplicabilidade real — trade-offs explícitos, robustez sob adversidade |
| V | Densidade Executiva | Compressão cognitiva — máxima densidade informacional |
| VI | Filosófico-Ontológico | Profundidade conceitual — pressupostos, implicações epistemológicas |
| VII | Crítico-Adversarial | Estresse intelectual — steel man, falsificação externa |
| VIII | Meta-Cognitivo | Controle da própria inferência — calibração, detecção de viés |
| IX | Abdutivo-Heurístico | Raciocínio pela melhor explicação — diagnóstico, Peirce |
| X | Síntese Dialética | Unificação de contradições — Hegel, aufhebung, paradoxo produtivo |
| XI | Colapso Epistêmico | Suspensão radical e reconstrução — epoché, primeiros princípios |

**Pipeline padrão:**
```
I Explorar → II Estruturar → III Conectar → VII Criticar → X Sintetizar → V Comprimir
```

**Perfis de combinação disponíveis:**
- Perfil Estratégico-Executivo: I + IV + VII + V
- Perfil Científico-Rigoroso: IX + II + VIII + VI
- Perfil Inovação-Radical: XI + I + X + VI
- Perfil Diagnóstico-Sistêmico: IX + III + II + VIII

### Config Agêntica — Camada 9 (`config-agentica/`)

A nona camada, ativada quando a complexidade epistemológica do problema exige não um único processo inferencial, mas uma **rede de agentes especializados** — cada um guardião de uma das três verdades, conectados por loops de retroalimentação.

#### As Três Verdades

O sistema agêntico é ancorado na distinção entre três tipos de verdade que nenhum agente singular pode maximizar simultaneamente:

| Verdade | Natureza | Agente Responsável |
|---------|----------|-------------------|
| **Probabilística** | O que os dados e evidências empiricamente indicam | Agente Estatístico |
| **Lógica** | O que é coerente com o sistema de regras e constraints vigentes | Agente Lógico |
| **Pragmática** | O que funciona no mundo real com suas fricções e limitações | Agente Pragmático |

#### O Protocolo de Retroalimentação

```
P(output | manifesto ∧ goal ∧ warnings ∧ constraints ∧ constructs
           ∧ contexto ∧ output_spec ∧ paramprompting)

Síntese(t) = Π( L( P(dados, t) ), constraints ) ∧ viável(mundo_real)

SE L( P(t) ) → contradição:  reinjetar com Ω(t+1) = pivotar(Ω(t), erro_detectado)
SE Π( síntese ) → inviável:  reinjetar com Ω(t+1) = pivotar(Ω(t), fricção_real)
```

A **contradição entre agentes não é falha** — é o mecanismo mais poderoso do sistema. É precisamente quando o Estatístico e o Lógico discordam que o Orquestrador encontra o espaço onde a solução genuinamente nova reside.

#### Domínios de Aplicação

A Config Agêntica é domínio-agnóstica. Exemplos de aplicação:

- **Inteligência Comercial** — demanda vs. margem vs. comportamento de compra real
- **Pesquisa Jurídica** — jurisprudência empírica vs. coerência normativa vs. defensabilidade processual
- **Segurança Ofensiva** — superfície de ataque vs. condições de exploração vs. viabilidade no ambiente real
- **Pesquisa Científica** — literatura empírica vs. coerência teórica vs. testabilidade metodológica
- **Decisão Estratégica** — sinais de mercado vs. consistência com tese estratégica vs. capacidade de execução
- **Produção Criativa** — corpus de ressonância cultural vs. coerência interna vs. recepção real do público
- **Interpretação Profunda** — padrões simbólicos vs. framework interpretativo vs. eficácia transformadora

### Orquestrador de Prompts (`orquestrador/`)

Como transformar qualquer chat de IA (ChatGPT, Claude, Gemini) em um motor epistêmico de investigação — usando o próprio Método Coragem™ como arquitetura do sistema.

**Princípio central:** um Orquestrador não cospe prompts prontos. Ele investiga — conduzindo o usuário através de três waves de perguntas que transformam uma intenção vaga numa arquitetura de condicionamento cognitivo precisa, antes de qualquer execução.

#### O Protocolo das 3 Waves

```
WAVE 1 — SUPERFÍCIE OPERACIONAL
Popula: Goal (C2) + Constraints (C4)
→ O que precisa existir como artefato?
→ Quem vai usar e em que contexto?
→ Quais os constraints imediatos?
→ Existe algum exemplo de referência?

WAVE 2 — CONTEXTO E PADRÕES
Popula: Contexto Intrínseco (C6) + Warnings (C3)
→ O que desencadeou esta necessidade agora?
→ O que já foi tentado? O que falhou?
→ Quem tem interesse e que posições adotam?
→ Que dados ou evidências estão disponíveis?

WAVE 3 — EPISTEMOLOGIA E CRITÉRIOS
Popula: Manifesto (C1) + Paramprompting (C8) + Output Esperado (C7)
→ Como saberia que o output foi bem-sucedido?
→ Posição clara ou múltiplas perspectivas em tensão?
→ Dados empíricos, lógica conceitual ou experiência prática?
→ O que definitivamente não deve aparecer?
→ Abertura para requalificação do próprio problema?
```

#### Como Ativar

**ChatGPT:** Configurações → Instruções personalizadas → campo "Como o ChatGPT deve se comportar?"  
**Claude:** Criar Project → Project Instructions  
**Gemini:** Criar uma Gem com o preset como instruções base  
**Qualquer LLM com API:** Inserir como `system` no array de mensagens

O preset completo está em `orquestrador/Orquestrador_MC_v4_Preset.txt`.

## Fundamento Teórico

O Método Coragem™ se apoia em referências de múltiplas tradições, não como ornamento, mas como fundação técnica:

**Filosofia da linguagem e fenomenologia**
- *Husserl* — epoché e intencionalidade como modelo para o Manifesto
- *Wittgenstein* — limites da linguagem como limites do espaço inferencial (Constructs)
- *Gadamer* — horizonte hermenêutico como modelo para o Contexto Intrínseco
- *Austin* — teoria dos atos de fala (locução/ilocução/perlocução) para o Output Esperado

**Epistemologia e filosofia da ciência**
- *Popper* — falsificacionismo interno no Modo VII (Crítico-Adversarial)
- *Kuhn* — anomalias como motores de inovação no Modo IX (Abdutivo)
- *Peirce* — lógica abdutiva como fundação do raciocínio pela melhor explicação

**Ciência cognitiva e teoria da informação**
- *Kahneman* — distinção Sistema 1/Sistema 2 como modelo para o Paramprompting
- *Shannon* — entropia e compressão sem perdas como modelo para o Modo V
- *Hegel* — dialética e aufhebung como fundação do Modo X (Síntese Dialética)

## Princípio Unificador

> *"O sistema que aprende com suas próprias contradições é epistemicamente superior ao sistema que as evita. A retroalimentação agêntica não é debugging — é a arquitetura pela qual verdades incompatíveis geram sínteses impossíveis de prever."*

A transição paradigmática que o Método Coragem™ propõe:

```
DE: Prompt como instrução textual para execução de tarefa
PARA: Prompt como arquitetura de condicionamento cognitivo multidimensional
```

Três níveis de controle simultâneos:
- **Operacional** — o que fazer (Goal + Output)
- **Epistêmico** — com qual verdade e valor (Manifesto + Constructs + Contexto)
- **Meta-Cognitivo** — como pensar sobre isso (Warnings + Constraints + Paramprompting)
